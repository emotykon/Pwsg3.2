Dodanie standardowego, szarego przycisku (kontrolki BUTTON) wewnątrz okna wymaga stworzenia tzw. okna podrzędnego (child window) przy użyciu predefiniowanej klasy systemowej "BUTTON".

Oto jak zmodyfikować projekt, aby po kliknięciu w fizyczny przycisk na środku ekranu wyskakiwało okienko koloru:

1. Zmiany w app_2048.h
Musimy dodać uchwyt (HWND) dla przycisku oraz jego unikalne ID.

C++
#define ID_MY_BUTTON 2001 // Unikalne ID przycisku

class app_2048 {
private:
    HWND m_button; // Uchwyt do przycisku
    // ... reszta pól ...
};
2. Tworzenie przycisku w app_2048.cpp
Przycisk tworzymy zazwyczaj w konstruktorze lub podczas obsługi komunikatu WM_CREATE. Używamy do tego CreateWindowExW z klasą L"BUTTON".

C++
app_2048::app_2048(HINSTANCE instance) : m_instance{ instance } {
    register_class();
    m_main = create_window();

    // Tworzymy przycisk jako dziecko okna głównego
    m_button = CreateWindowExW(
        0,
        L"BUTTON",             // Systemowa klasa przycisku
        L"Wybierz Kolor",      // Napis na przycisku
        WS_TABSTOP | WS_VISIBLE | WS_CHILD | BS_DEFPUSHBUTTON,
        20, 20,                // Pozycja X, Y
        150, 40,               // Szerokość, Wysokość
        m_main,                // Rodzic (nasze główne okno)
        (HMENU)ID_MY_BUTTON,   // ID przycisku (rzutowane na HMENU)
        m_instance,
        nullptr
    );
}
3. Obsługa kliknięcia w window_proc
Kiedy użytkownik kliknie przycisk, system wyśle do głównego okna komunikat WM_COMMAND. Obsługa jest identyczna jak w przypadku menu, zmienia się tylko sprawdzane ID.

C++
LRESULT app_2048::window_proc(HWND window, UINT message, WPARAM wparam, LPARAM lparam) {
    switch (message) {
    case WM_COMMAND: {
        // Sprawdzamy, czy kliknięto nasz przycisk
        if (LOWORD(wparam) == ID_MY_BUTTON) {
            CHOOSECOLORW cc{ sizeof(cc) };
            static COLORREF cust_colors[16];
            cc.hwndOwner = window;
            cc.lpCustColors = cust_colors;
            cc.Flags = CC_FULLOPEN | CC_RGBINIT;
            cc.rgbResult = m_color1;

            if (ChooseColorW(&cc)) {
                m_color1 = cc.rgbResult;
                InvalidateRect(window, nullptr, TRUE);
            }
        }
        return 0;
    }
    // ... reszta case-ów ...
    }
    return DefWindowProcW(window, message, wparam, lparam);
}
Jak to działa?
Klasa "BUTTON": Windows ma wbudowane "przepisy" na standardowe kontrolki (przyciski, pola tekstowe, listy). Nie musisz rejestrować klasy dla przycisku – system już ją zna.

WS_CHILD: Ten styl sprawia, że przycisk jest "przyklejony" do Twojego okna. Porusza się razem z nim i znika, gdy zamkniesz program.

Parametr hMenu: W funkcji CreateWindowEx, gdy tworzysz okno podrzędne (WS_CHILD), parametr, który normalnie służy do przekazywania menu, służy do przekazania identyfikatora (ID) kontrolki. To dzięki temu w WM_COMMAND wiemy, że kliknięto akurat ten przycisk.

Dlaczego to wyskakuje tak samo?
Niezależnie od tego, czy wywołasz ChooseColorW z menu, z przycisku, czy nawet po naciśnięciu klawisza na klawiaturze – mechanizm "wyskakiwania" jest zawsze ten sam. Różni się tylko wyzwalacz (trigger) w Twoim kodzie.
